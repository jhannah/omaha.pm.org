I hope a more experienced perl programmer can help me understand where I
went wrong here.

I am reading in a file that has two values on each line separated by a
tab.  Here is the code I was using to get at this data:

while ( <IN> ) {
  chop;
  ($filename,$directory) = split(/\t/);
  $directories{lc($filename)} = $directory;

  # make a list of all the different directories we come across in the
file
  if (length($directory)) {
    $uniquedirectories{$directory}++;
    }
  }
close(IN);

Then when I wanted a list of all the unique directories I would use:
foreach $directory (sort keys %uniquedirectories) {
  print "this is the directory:($directory)\n";
  }

The problem is my output always looked like this:
)his is the directory:(
)his is the directory:(ALM
)his is the directory:(Alumni
)his is the directory:(COP
)his is the directory:(Careers
)his is the directory:(Germany
)his is the directory:(Graduates

Which I can't figure out.  Where did the T in 'This' go?  I finally
realized it had something to do with the newlines in the file.  I fixed
it by using "$_ =~ s/\r\n//;"  instead of "chop;".   But what was going
on?  The file did come from a windows system, so I thought maybe chop
was just pulling off the newline but not the CR? So I tried chomp,
chomp;chomp;, and chop;chop; but nothing helped.

Thanks for any help.
