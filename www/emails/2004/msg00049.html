On Friday, February 6, 2004, at 11:06 AM, Mike Hostetler wrote:
Are there any PerlScript hackers out there?

Not me. But my ignorance never stops me from giving half baked advice. -grin-

We want to get at objects that are availiable via the MS Windows
Scripting Host.  We can get at them with VBScript and JScript, but we
cannot with PerlScript.  I've tried the following:

				o my $var = $Session;
				o my $var = Session;
				o my $var = ${Session};

And every time I get the same thing::

Description: Global symbol "$Session" requires explicit package name at
(eval 2) line 2.
Global symbol "$Session" requires explicit package name
Line: 2' {F:\develop\wp32\WpMTS\ScriptActive.cpp, 245}

Anyone have any ideas? I don't think I should have to do:

				use Win32::OLE;
				my $session = Win32::OLE->new("WorkPoint.wpSession");
every time, since I don't have to do it with VB.

-ponder- It seems to me that Perl is doing the Right Thing which is to avoid polluting your namespace with junk until you explicitly tell it to do so. Two former lives ago I coded ASP for a living and I remember having all kinds of stuff at my fingertips on line 1 of my program. I think as a general rule that won't be true in Perl environs.

Under Apache mod_perl I believe you can put arbitrary stuff in Apache startup (httpd.conf) which will be in your namespace, but you probably didn't care about that. -grin- Perhaps there's a PerlScript equivalent.

Maybe you could "hide" the ugliness in another package. Mike.pm:

   package Mike;
   use Win32::OLE;
   sub session {
      return Win32::OLE->new("WorkPoint.wpSession");
   }

Then to use that, you'd:

   use Mike;
   my $session = Mike::session;

That pattern isn't too handy unless you have a bunch of different Win32::OLE+ thingies you want to grab easier than the default way in a bunch of different programs...?

Ok, I'll shut up now. -grin-
