Ryan wrote:
> I want the browser to ask the user for username and password input, but
> instead it just times out with these error messages showing up in my
> Apache error_log:
> stty: standard input: Invalid argument
> stty: standard input: Invalid argument
> login failed: bad name or password at
> /usr/local/apache/cgi-bin/telnet.pl line 29
[snip]
> The code:
> ------- Begin 'telnet.pl' -------
[snip]
> print "Enter the username for the $HOST account: ";
> $username = <STDIN>;
> chomp $username;

I believe the problem is that there is no "<STDIN>" available from a Perl
CGI script -- i.e. there is no user interaction with the script until the
script has completed and closed the output stream.

As a test, if you hard-coded the $username and other information, does it
log in and send back what you want?  You'll have to write your script so
that it runs in 'chunks'.  I.E. the first run would put the HTML form
information up for the Username and password with a "submit" button.  The
submit button would then call the script with the Username and Password
variables set from the form, and continue executing.

It will be tricky if you are trying to make an HTML interface to an
interactive program, but I believe I have seen some implementations done
before.  Basically, you will take in some input (a command to run) in a
form and have the user "submit" it, let the command run and capture the
output, send that output back to the user in a new form and wait for
another command.  You'll probably also want/need to setup some sort of
session handlling on the server side.

Good luck!
