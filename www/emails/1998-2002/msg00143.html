map *is* a mind bender in the beginning.

Breaking it into conceptual chunks helps, I think:
1. The Syntax
2. The Operation
2. The Context


The Syntax

straight out of perlfunc for map:

map BLOCK LIST 
map EXPR,LIST

Evaluates the BLOCK or EXPR for each element of LIST (locally setting $_
to each element) and returns the list value composed of the results of
each such evaluation.  Evaluates BLOCK or EXPR in a list context, so
each element of LIST may produce zero, one, or more elements in the
returned value.


The Operation

The function always operates on a list of input (an input list? :), so
whenever you see 'map', first think about the list that it is operating
on.  For example, using a variation of Tim's example:

map { print substr($_, 0, 3) . "\n"; } @Array;

This map function is operating on the list represented by '@Array'. This
is the input to the 'map' function in this example.

OK, now that we have a list of input, now we can do something with it. 
What we do with the input is the part is represented in the BLOCK/EXPR
portion of the 'map' syntax.  A good way to think of the BLOCK/EXPR
portion of the function is to consider it as code you would find in a
'foreach' loop.  You've got a list of input (represented by @Array) and
you want to do something with each element of that list.  This is the
code that you put in the BLOCK/EXPR portion of the 'map' function.

For example, the following two snippets do the same thing:

# 'map' snippet
map { print substr($_, 0, 3) . "\n"; } @Array;

# 'foreach' snippet
foreach (@Array) {
    print substr($_, 0, 3) . "\n";
}

In both cases, each element of @Array is loaded into the 'default input
space' variable '$_', and then used in the '{ BLOCK }' of code.  So up
to this point, 'map' is really just like an odd way of doing a 'foreach'
loop ... putting the block of code first, and then trailing it with the
list that will be used as input to that block.


The Context

Now for the really powerful part of 'map' ... the context in which it is
called. In the example shown above, the 'map' function is called in a
void context ... we are not trying to assign its output to a variable of
some sort, or looking for its return value, or anything like that.  As
shown, when called in a void context, 'map' is really pretty much just
an odd way of writing a 'foreach' loop.

But called, for example, in a list context, the 'map' function can be
used to do all kinds of neat things, such as transformations. Stealing
from Tim's examples, again, consider:

@Array = map { substr($_, 0, 3) } @Array;

This short little line of Perl code converts the original @Array
elements into the first three letter abbreviation of each of the former
elements ... pretty cool! ... and, conceptually, pretty easy to
understand, once you have the proper "Perl mind-set" !
