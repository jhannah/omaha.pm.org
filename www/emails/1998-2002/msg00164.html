I currently work in the Midrange support organization at Qwest. I just
joined a team that is undertaking the implementation of a suite of
system monitoring tools written completely in Perl, with the flexibility
to enable support people to add in their own tools written in whatever
language they like to be managed by this core set of tools. I've also
had the privilege of developing a new knowledge base repository website
on an HP system using Perl/CGI exclusively (so far) which will be used
by the support folks.

Perl has been in use here for a long time from what I can gather, long
before I ever set foot in the door for the first time. I have been here
almost two years (from the classic US West organization, not classic
Qwest) and 4 months before I started here full-time, ended a year stint
as a contractor (US West). What with the merger and all, we've been
going through the usual consolidating systems and trying to move toward
more of a corporate standard environment from the previous free-for-all
that it had been. At this point it's unclear how things will look in a
few years, if Perl will still be as prevalent here as it was pre-merger
(USWest) and currently.

Okay someone else's turn to ramble... :)
