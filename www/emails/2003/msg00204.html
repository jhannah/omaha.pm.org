Hi Graham --

I stumbled into this today. Reading the POD for Net::FTP I assumed I could
make calls like this:

   $ftp->type('ascii');
   $ftp->type('binary');

But I can't. The correct TYPE values are not "ascii", etc., but are
actually "A", "I", etc. I didn't see that explained in the POD anywhere.
So, I thought I send this patch to you to see if you'd be interested in
applying it.

Cheers,

Jay Hannah
Omaha Perl Mongers: http://omaha.pm.org



--- FTP.pm      Thu Sep 11 12:31:04 2003
+++ FTP_new.pm  Thu Sep 11 12:28:18 2003
@@ -1366,7 +1366,12 @@

 This method will send the TYPE command to the remote FTP server
 to change the type of data transfer. The return value is the previous
-value.
+value. Valid TYPEs are:
+
+  $ftp->type('A');   # ascii
+  $ftp->type('I');   # binary
+  $ftp->type('E');   # ebcdic
+  $ftp->type('L');   # byte

 =item ascii ([ARGS]) binary([ARGS]) ebcdic([ARGS]) byte([ARGS])
