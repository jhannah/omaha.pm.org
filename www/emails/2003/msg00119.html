I suppose you could munge the array into a single delimited string and use a
regex on it. Bonus points for using regexs, but not overly efficient.

You could try sorting the array if $bad was close to the top or bottom of
the array. Assumes a lot.

Last is your friend too. In this case it could save 0 to 99 iterations of
the loop, depending on where bad is in the array. Again, works best if $bad
is found early.

    	$flag = 0;
	foreach $item (@array) {
	if ($item eq $bad) {
		$flag = 1;
		last;
		}
	}
	if ($flag) {
		## do something
	}

Being completely evil, if the values in the array must be unique, why not
use a hash where the key and the value are the same? That makes finding
element $bad as easy as:

If ($arry{$bad} eq $bad) {
	### do something
}

Double bonus points for using hashes. <grin>
