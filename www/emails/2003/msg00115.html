thehaas@xxxxxxxxxx wrote:
> Is there anyway I can find out if a scalar value I have is an element of
> an array without looping through the whole array looking for it?

Well, in general you want to:
   - Build an array if you want to look something up by position.
   - Build a hash if you want to look something up by key.

So the first question I should ask is: Since you're wanting to do a
lookup up by key, should you have built a hash instead of an array?

Assuming the answer is no, for whatever reason, you're going to have to
"loop the array". There are shortcuts, if you want them.

my @a = (1, 2, 3, "find me!", 4, 5, 6);

The most straight forward way:

foreach (@a) {
   if ($_ eq "find me!") {
      # Do whatever
   }
}

Shortcuts:

if (grep /^find me!$/, @a) {
   # Do whatever
}

if (grep {$_ eq "find me!"} @a) {
   # Do whatever
}

if ((join "|", @a) =~ /\|find me!\|/)) {
   # Do whatever
}

(and many more I can't think of off the top of my head, I'm sure...)

Or, if you're feeling really saucy, you could play around with any of
the CPAN libraries which give you constructs that allow both position
and key lookups simultaneously. I've never used any such constructs, so
I can't recommend any specifically without diving through some books.

HTH,
