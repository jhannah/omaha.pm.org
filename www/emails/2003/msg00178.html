On Thu, Aug 21, 2003 at 09:38:35AM -0500, Mike Hostetler wrote:
> 
> I'm writing a script that looks at time frames,  and I get a message
> every time I run it like this:
> 
> Subroutine main::ctime redefined at /usr/lib/perl5/5.8.0/Exporter.pm line 59.
>  at /usr/lib/perl5/5.8.0/i686-linux/POSIX.pm line 19
> Prototype mismatch: sub main::ctime (;$) vs none at
> /usr/lib/perl5/5.8.0/Exporter.pm line 59.
>  at /usr/lib/perl5/5.8.0/i686-linux/POSIX.pm line 19
> 
> But the script does run correctly.
> 
> I've narrowed down the cause to the "use" statements:
> #################
> #!/usr/bin/perl -w
> 
> use Time::Local;
> use Time::localtime;
> use POSIX;
> ###############3

Both POSIX.pm and Time/localtime.pm are defining and exporting sub ctime() (one
with a prototype, one without). 

According to Time::localtime's pod docs, you can pass it an empty export
list to play more diplomatically with the global namespace.

use Time::localtime qw();
use POSIX;

But, from there on out you'll need to prefix Time::localtime package
name to all of its subroutines. So, for example, you'll need to call
Time::localtime::asctime instead of asctime (or whatever you're using
Time/localtime.pm for).
