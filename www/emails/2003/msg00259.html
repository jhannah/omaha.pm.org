Ahhh... From "perldoc perlrun"...

       -0[digits]
            specifies the input record separator (`$/') as an
            octal number.  If there are no digits, the null char-
            acter is the separator.  Other switches may precede
            or follow the digits.  For example, if you have a
            version of find which can print filenames terminated
            by the null character, you can say this:

find . -name '*.orig' -print0 | perl -n0e unlink

            The special value 00 will cause Perl to slurp files
            in paragraph mode.  The value 0777 will cause Perl to
            slurp files whole because there is no legal character
            with that value.

So that's what -00 does! How many quirky shortcuts can you pack into one language? -grin-

I use this pretty regularly these days, mostly as a 'multiline grep' in one-liner form:

$ perl -00 -ne 'print if /interesting stuff/' somefile

Most RADIUS servers log their flat file accounting records in a multiline format separated by blank lines, and we have a couple of in-house apps that log messages that way (e.g., Java stack dumps).

Question:
How can I read in a file by paragraphs?

    Use the `$\' variable (see the perlvar manpage for details). You
    can either set it to `""' to eliminate empty paragraphs
    (`"abc\n\n\n\ndef"', for instance, gets treated as two paragraphs
    and not three), or `"\n\n"' to accept empty paragraphs.
